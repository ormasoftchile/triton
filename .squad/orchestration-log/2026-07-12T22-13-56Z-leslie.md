## Options Analysis & Recommendation

### Icons — Recommendation: Phase 1 Implementation

**Option A: Bundled inline-SVG icon registry** (RECOMMENDED)
- Registry of 24×24 viewBox icons as path primitives
- Deterministic, offline, PNG-safe
- Reuse `scene-icon-glyph` SKILL pattern (translate/scale formula)
- Baseline: 50–100 core icons; extensible

**Option B: User-supplied SVG icons** (Phase 3)
- Load external SVG files, inline as `<path>`
- Requires validation, collision detection
- Deferrable after Phase 1

**Option C: Font-Awesome webfont** (REJECTED)
- Requires system font or CDN
- Incompatible with offline static PNG

### Images — Recommendation: Deferred (Phase 2–3)

**Constraints:**
- rsvg-convert: No `<foreignObject>` support
- Only inline SVG or data: URIs safe
- Local file embedding requires upfront validation

**Options:**
- Data-URI embedding (Phase 2)
- Local file registry (Phase 3)
- Remote URLs (never — breaks determinism)

### Notes — Recommendation: Phase 1 Overlay Extension

**Attached notes** (`note on <element>: "text"`):
- Map to `RawNote` → overlay system
- Theme-aware rendering (palette colors)
- De-collision via `chromeRects`

**Free-floating notes** (`note "text" [at position]`):
- Generalize poster `PosterNote` to all diagram types
- Corner-based positioning (Phase 1); coordinate-based (Phase 2)
