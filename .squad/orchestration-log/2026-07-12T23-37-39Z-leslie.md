## Two Attachment Forms

### Node-Shape Icon
Icon decorates a node/box/cell as a visual glyph. Grammar: `@{ icon: "prefix:name" }` metadata block (for grammars without dedicated icon slot) or existing `(icon)` / `::icon(...)` syntax (architecture, mindmap).

### Inline-Label Icon
Icon embedded inline within text/label. Token: `:prefix:name:` (colon-wrapped, unambiguous against quoted strings and `%%` comments).

## IR Design (Mark's domain — deferred to implementation phase)

**IconRef type:**
```typescript
export interface IconRef {
  readonly prefix: string;   // e.g. "azure"
  readonly name: string;     // e.g. "app-service"
}
```

Shared across all diagram IRs. One resolver function `resolveIcon(registry, ref)`. Host provides `ResolvedIconRegistry`; core stays pure.

---

## Mermaid Compatibility

Adopts `@{ icon: "..." }` syntax where Mermaid v11+ defines it (flowchart shape metadata). Triton extends to other grammars. The `:prefix:name:` inline form is Triton-original (Mermaid's `fa:fa-x` is htmlLabels-only, unsuitable for static SVG).
