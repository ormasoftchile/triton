## Notable Learnings

### CI Brittleness Root Causes (Phase 4–5)

Three cascading failures in `test/latex-cli-theme.test.ts`:

1. **Test spawned non-existent dist/:** `latex/dist/cli.cjs` doesn't exist at test time (CI runs `pnpm test` before `pnpm build`).
   - Fix: Extract and export the function; test imports TS source.

2. **Module entry point runs on import:** `cli.ts` called `main()` at the top level.
   - Fix: Guard with `import.meta.url === process.argv[1]` (ES module equivalent of CommonJS `require.main === module`).

3. **Transitive pdfkit import:** Importing from `cli.ts` pulls `pdfkit` (latex-only dep); CI doesn't install latex dependencies.
   - Fix: Move the CLI-independent theme-resolution logic into a pure `theme-resolve.ts` that depends only on core (node + ../../src/**).

**Pattern to Avoid:** `vi.spyOn(process, 'exit')` — vitest intercepts process.exit globally; reports an uncaught error even when the spy is set up correctly and the test passes.

### Phase 5 Schema Discovery

`$schema` field rejected by strict validator. Users can still get editor hints via `json.schemas` in VS Code settings, referencing the canonical schema URL. Template omits `$schema` to remain valid.
