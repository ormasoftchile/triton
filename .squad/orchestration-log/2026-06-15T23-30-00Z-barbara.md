## Key Work

### Finding A: Multi-Line Node Labels Are NOT Supported

**Root Cause:** Flow grammar layout (`packages/core/src/grammars/flow/layout.ts`) and tree grammar layout (`packages/core/src/grammars/tree/layout.ts`) emit node labels as `kind: 'text'` primitives with raw label string. The `extractLabel` function in flow parser (flowchart.ts:794) strips quotes but does NOT interpret `\n` or `<br>`. No code path reaches `MultiTextPrimitive` for node labels.

**Impact:** Authors writing `A["line1\nline2"]` expecting a line break see literal `line1\nline2` — silent authoring failure, no warning emitted.

**Classification:** Tier-2 feature gap; not a blocker for shipping.

### Fix A: theme-contract.mmd

**Problem:** Seven node labels used `\n` as line-break separator; rendered as literal backslash-n.

**Solution:** Single-line equivalents:
- Hub node: `"Theme Contract\npalette · fonts\nspacing · density\nshape language"` → `"Theme Contract"`
- Binding nodes: `"bindFlowTheme\n→ FlowTheme"` → `"bindFlowTheme → FlowTheme"` (all six)

**Result:** 1033×550 px (scale-1), aspect 1.88:1, height 550 — within dimension guard. Clean and professional.

### Fix B: family-taxonomy.mmd

**Problem (A):** Root label `root((5 Diagram\nFamilies))` rendered as `5 Diagram\nFamilies`.

**Problem (B):** UML branch leaves (`classDiagram`, `sequenceDiagram`, `stateDiagram`, `erDiagram`) placed by radial mindmap layout in tight bottom-left sector — labels overlapping and unreadable.

**Solution:** 
1. Root: `"5 Diagram\nFamilies"` → `"5 Diagram Families"`
2. UML leaves shortened to single-word aliases: `class`, `sequence`, `state`, `ER`

**Result:** 1400×1000 px (scale-1), aspect 1.4:1, height 1000 — within dimension guard. All families and diagram types visible, readable, no overlap.
