## Outcome
Identified the **SVG→PDF format gap** (Triton emits SVG; `\includegraphics` never accepts SVG → converter required).
Recommended **precompile + `\includegraphics`** via a thin `triton.sty` driven by a CLI; rejected inline
shell-escape (Overleaf-hostile) and the svg-package/Inkscape route (heavy per-machine dep). **Flagged that no
Triton CLI existed** (no `bin`/`main`/`exports`) — a hard prerequisite. Posed 5 open questions to the user
(vector PDF vs PNG, Overleaf hard requirement, inline vs precompile-only, add a CLI, native PDF backend),
which the user then ratified into the Phase-2 constraints.
