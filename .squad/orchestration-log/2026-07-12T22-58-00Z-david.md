**File:** `.squad/decisions/inbox/david-icon-library-format-research.md`

**Summary:**
- **Azure Architecture Icons:** Verified 705+ official icons from Microsoft. Located at `https://arch-center.azureedge.net/icons/Azure_Public_Service_Icons_V24.zip`. Multicolor (hardcoded brand fills: `#0078D4`, teal, gradients). Monthly updates.
- **Licensing Analysis:** Microsoft permits use "in architectural diagrams, training materials, or documentation" only. Third-party tool bundling = redistribution outside permitted use. HARD CONSTRAINT: Triton cannot ship Azure icons. BYOP model required.
- **Iconify Coverage Gap:** No `@iconify-json/azure` pack exists (July 2026). Confirmed via icon-sets GitHub repo. Only ~9 generic Azure brand marks scattered across `logos:`, `simple-icons:`, `devicon:` — NOT the 705+ service icons. Iconify cannot fill this gap due to Microsoft licensing.
- **Format Selection:** Evaluated IconifyJSON (`@iconify/types@2.0.0`), directory-of-SVGs, and SVG sprite alternatives. IconifyJSON wins: single artifact, standard schema, load-once/lookup, ecosystem tooling (Iconify publishes 150+ packs in this format).
- **File Size Measurements:**
  - `@iconify-json/mdi` — 7,638 icons, 3.0 MB
  - `@iconify-json/logos` — 2,091 icons, 7.4 MB
  - Azure (estimated) — 705 icons, 2–5 MB
  - All acceptable for compile-time load-once tolerance.
- **Conversion Pipeline:** `@iconify/tools` `importDirectory()` + `cleanupSVG()` proven recipe. Do NOT call `parseColors()` for brand icons (preserves fills). Normalizes SVG metadata. User-runnable once per Azure release.

### Monochrome vs Multicolor Classification

**Key finding:** Icon classification deterministic at pack-load time by scanning body content for hardcoded fills.
- **Monochrome:** body uses `fill="currentColor"` only → palette-tintable.
- **Brand/Multicolor:** body contains `fill="#..."` or `fill="url(#...)"` (gradients) → rendered verbatim.
- **Azure reality:** ALL 705+ icons are brand/multicolor with Microsoft-designed gradients. Cannot be tinted to dark theme; must render as-is.
