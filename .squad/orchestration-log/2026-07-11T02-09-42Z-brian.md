## Root-Cause Learning

**Issue:** Initial implementation embedded JSON `<script>` tag inside SVG string. VS Code webview innerHTML parsing + CSP blocked the script, causing blank preview.

**Root cause:** CSP + innerHTML don't coexist with new script tags. The manifest must travel as a separate payload, not inside SVG content.

**Resolution:** `compileAndRenderSync` now returns `{ svg, anchors }` separately. Extension posts both via `vscodeApi.postMessage()` with untrusted content isolated.

**Lesson:** Webview-panel-must-be-closed-to-refresh — vs-code extension webview state caching means changes to the rendering pipeline require the user to close and reopen the side panel for new code to take effect during dev. This was discovered during live-testing regressions.
