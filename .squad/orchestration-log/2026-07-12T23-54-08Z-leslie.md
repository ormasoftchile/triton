## Work Summary

Evaluated two syntactic approaches for attaching node annotations to Triton flowchart nodes:

1. **`@key:value` connector-style** (Triton-native, extends existing edge annotation sigil)
2. **`@{...}` object form** (Mermaid v11+ compat)

### Key Findings

- **`@` is already Triton's annotation sigil** (edges: `grammar.peggy:147`). Extending to nodes unifies dialect.
- **`@icon:azure:app-service` parses unambiguously** — key is first segment (before first `:`), value is greedy remainder.
- **No positional ambiguities** — node annotations appear after shape/label brackets (distinct from edge annotations).
- **Grammar simplicity favors `@key:value`** — no JSON-subset parser needed.
- **`@{...}` can coexist as parse-time alias** — both lower to same IR.

### Recommendation

**CANONICAL syntax:** `@key:value` (Triton-native)  
**ALIAS (Mermaid compat):** `@{...}` → same IR lowering

### Example (final card-node syntax)

```
flowchart LR
  A ["App Service"] @shape:card @icon:azure:app-service
  B ["PostgreSQL\nPrimary data store"] @shape:card @icon:mdi:database
  A -->|queries| B @orthogonal:EW
```
